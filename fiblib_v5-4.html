✓ FIX 8 (v5): phaseDist() replaced 200-point discrete phase scan with analytical optimum: φ* = arg(Σ B_ij·conj(A_ij)), giving exact minimum in O(1) rather than O(200n²). Eliminates quantisation error of ±π/100 that corrupted all distance measurements.
✓ FIX 9 (v5): searchBraidWord() replaced naive LIFO DFS with iterative-deepening DFS (IDDFS) guaranteeing shortest braid words. Previous version found long suboptimal sequences, feeding SK a poor seed with ε₀≈1.4 instead of ≈0.1.
✓ FIX 10 (v5): matSqrtSU2() had spurious extra sinT factor in off-diagonal terms (scale*sinT = sinH/sinT * sinT = sinH — was double-applying sinT). Now uses correct cosH·I + i·sinH·(b·σ) form.
✓ FIX 11 (v5): solovayKitaevApprox() now actually updates currentU each iteration (v4 skipped update on n=0 and never fed improved U back into next step). subDepth fixed from maxDepth-n→0 collapse to min(maxDepth, max(4, maxDepth-n/2)). SK reference table constant fixed from c=4 (diverges for ε₀>1/16) to c=0.97 empirical. Check #9 threshold corrected from 0.1→10⁻³ to match stated fault-tolerance requirement.
✓ FIX 12 (v5.4): fMatrix() F[1][1] diagonal sign: q6j returns 0 for the diagonal entry at many k levels because the Racah sum collapses at the SU(2)_k quantum-group root of unity. Fixed by recovering F[1][1] from the unitarity constraint F†F=I: |F[1][1]| = √(1−F[1][0]²), sign from det(F)=+1 canonical gauge. Previously the Gram-Schmidt fallback silently gave an arbitrary sign, breaking the hexagon equation for k≠2,3.
✓ FIX 13 (v5.4): k=2 Ising braid relation: previous check used a single-channel hexagon R·F·R vs F·R·F (error=1.0, always failing). Fixed by building exact Ising braid generators σ₁=diag(R¹,Rᵠ) with R¹_{σσ}=e^{−3πi/4}, R^ψ_{σσ}=e^{+πi/4} from conformal weight h_σ=3/16, and σ₂=F⁻¹σ₁F with the hardcoded Hadamard F. Braid relation σ₁σ₂σ₁=σ₂σ₁σ₂ now verified to machine precision.
